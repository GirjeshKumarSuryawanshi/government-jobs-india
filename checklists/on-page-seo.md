# On-Page SEO Checklist

## Introduction

On-page SEO focuses on optimizing the content and structure of individual pages so they are useful, understandable, and easy to navigate.

Unlike technical SEO, on-page SEO emphasizes the quality of information, content organization, readability, and user experience.

This checklist provides practical recommendations that can be applied before publishing or updating a page.

---

# Search Intent

Before writing:

- [ ] Identify the primary user intent.
- [ ] Understand what users expect from the page.
- [ ] Choose the appropriate page type.
- [ ] Focus on solving the user's problem.
- [ ] Avoid creating pages without a clear purpose.

Examples of search intent:

- Informational
- Navigational
- Transactional
- Educational

---

# Page Title

- [ ] Unique title.
- [ ] Clearly describes the page.
- [ ] Important keywords appear naturally.
- [ ] Title accurately reflects the content.
- [ ] Avoid keyword stuffing.

Example

Good

```
SSC CGL 2026 Notification – Eligibility, Vacancy & Apply Online
```

Poor

```
SSC SSC CGL Jobs Government Jobs Notification Latest SSC Vacancy
```

---

# Meta Description

- [ ] Summarizes the page accurately.
- [ ] Encourages users to click.
- [ ] Matches page content.
- [ ] Written naturally.
- [ ] Unique for each important page.

---

# URL

- [ ] Short
- [ ] Readable
- [ ] Lowercase
- [ ] Hyphen-separated
- [ ] Descriptive

Example

```
/ssc-cgl-2026-notification
```

---

# Heading Structure

- [ ] One H1.
- [ ] Logical H2 sections.
- [ ] H3 headings where appropriate.
- [ ] Headings describe content clearly.
- [ ] Heading hierarchy is consistent.

---

# Introduction

The opening section should:

- [ ] Explain the topic.
- [ ] Answer the main question.
- [ ] Set user expectations.
- [ ] Avoid unnecessary filler.

---

# Content Quality

- [ ] Accurate information.
- [ ] Original writing.
- [ ] Easy to understand.
- [ ] Helpful explanations.
- [ ] Clear language.
- [ ] Updated where necessary.

---

# Readability

- [ ] Short paragraphs.
- [ ] Bullet lists used where helpful.
- [ ] Numbered steps when appropriate.
- [ ] Tables used for comparisons.
- [ ] Important information highlighted naturally.

---

# Recruitment Information

For Government Job pages, verify:

- [ ] Organization name.
- [ ] Notification title.
- [ ] Important dates.
- [ ] Eligibility.
- [ ] Age limit.
- [ ] Vacancy details.
- [ ] Salary.
- [ ] Application fee.
- [ ] Selection process.
- [ ] Required documents.
- [ ] Official notification link.
- [ ] Official application link.

---

# Internal Linking

- [ ] Link to parent category.
- [ ] Link to related recruitment.
- [ ] Link to syllabus.
- [ ] Link to admit card.
- [ ] Link to results.
- [ ] Link to previous papers.
- [ ] Link to official resources where appropriate.

---

# Images

- [ ] Descriptive filename.
- [ ] Alt text added.
- [ ] Optimized size.
- [ ] Appropriate format.
- [ ] Relevant to the page.

---

# Tables

Use tables where they improve clarity.

Examples:

- Eligibility
- Important Dates
- Vacancy Distribution
- Application Fees
- Salary

---

# Frequently Asked Questions

Include FAQs when users commonly ask similar questions.

Examples:

- Who can apply?
- What is the last date?
- Is there an application fee?
- What is the salary?
- How do I apply?

---

# Structured Data

- [ ] Appropriate schema selected.
- [ ] JSON-LD validated.
- [ ] Structured data matches visible content.
- [ ] Required properties reviewed.

---

# Mobile Experience

- [ ] Text readable.
- [ ] Buttons accessible.
- [ ] Tables responsive.
- [ ] Images scale correctly.
- [ ] No horizontal scrolling.

---

# Accessibility

- [ ] Descriptive headings.
- [ ] Alt text.
- [ ] Semantic HTML.
- [ ] Sufficient color contrast.
- [ ] Keyboard accessibility reviewed.

---

# E-E-A-T Considerations

Review whether the page demonstrates:

- [ ] Experience
- [ ] Expertise
- [ ] Authoritativeness
- [ ] Trustworthiness

Examples:

- Reference official notifications.
- Keep information current.
- Avoid unsupported claims.
- Distinguish facts from editorial commentary.

---

# AI Readability

Content should be easy for both users and automated systems to understand.

- [ ] Clear headings.
- [ ] Consistent terminology.
- [ ] Context before abbreviations.
- [ ] Logical structure.
- [ ] Helpful summaries.

---

# GEO Considerations

- [ ] Topic clearly defined.
- [ ] Entities identified.
- [ ] Context provided.
- [ ] Official sources referenced.
- [ ] Related topics connected.

---

# AEO Considerations

- [ ] Main question answered early.
- [ ] FAQ included where useful.
- [ ] Tables used appropriately.
- [ ] Answers are concise and complete.
- [ ] Headings match user questions.

---

# Before Publishing

- [ ] Title reviewed.
- [ ] Meta description reviewed.
- [ ] URL finalized.
- [ ] Images optimized.
- [ ] Internal links checked.
- [ ] External links verified.
- [ ] Structured data validated.
- [ ] Mobile tested.
- [ ] Proofreading completed.

---

# After Publishing

- [ ] Confirm page loads correctly.
- [ ] Verify internal links.
- [ ] Update XML Sitemap if required.
- [ ] Monitor indexing.
- [ ] Refresh information when official updates are released.

---

# Common Mistakes

Avoid:

- Keyword stuffing.
- Duplicate content.
- Misleading titles.
- Broken links.
- Outdated information.
- Missing headings.
- Thin content.
- Excessive repetition.
- Hidden text.

---

# Monthly Content Audit

Review:

- [ ] Outdated recruitment information.
- [ ] Broken links.
- [ ] Internal linking opportunities.
- [ ] FAQ improvements.
- [ ] Metadata updates.
- [ ] Structured data validation.
- [ ] Content freshness.
- [ ] User feedback.

# Conclusion

Good on-page SEO is about creating content that is accurate, well-organized, easy to understand, and genuinely useful.

Rather than optimizing for algorithms, focus on helping users find the information they need quickly and confidently.
